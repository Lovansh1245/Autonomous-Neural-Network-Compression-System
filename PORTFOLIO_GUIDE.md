## 🧠 PART 4: Interview Preparation

### The 30-Second Elevator Pitch
"I built a production-grade AI system that solves edge deployment bottlenecks. I engineered a custom PyTorch architecture that dynamically prunes its own tensor connections during training. An asynchronous FastAPI backend evaluates the hardware telemetry and passes it to an internalized FAISS RAG agent, which mathematically isolates the optimal configuration and pushes it to an interactive React dashboard for deployment."

### The 1-Minute Deep Dive
"Deploying dense networks to tight edge devices costs massive compute. To fix this, I engineered an algorithmic compression system. Instead of manual trial-and-error pruning, I injected continuous, learnable gating functions onto deep multi-layered architectures. The system calculates a dynamically bounded L1-Sparsity penalty pushing the architecture across a Pareto frontier tradeoff. The PyTorch loop produces hard-pruned physical configurations, which are logged to a FastAPI vector microservice. By integrating a local RAG agent, the system functionally reasons about its own telemetry, evaluating latency vs accuracy drop-offs to autonomously isolate the perfect deployment architecture."

### "What makes this unique?"
"Most researchers simply drop zero-values into a matrix using binary masks during 'pruning'. That doesn't reduce physical hardware allocations. My system executes **true physical structural pruning**. It slices the PyTorch arrays, deleting target sub-tensors permanently, which yields *actual* millisecond latency drops. Furthermore, I closed the loop by building a native analytical AI agent directly into the dashboard that queries those results using vector-math semantics."

### Deep Dive Q&A

**Why does L1 induce sparsity?**  
"Unlike L2 regularization which pushes weights closer to zero spherically, L1 regularization penalizes the absolute value of the vector uniformly. This geometrical boundary forces weaker gradients aggressively into exactly zero during gradient descent, forcing physical node dropouts."

**Why decouple Weight Decay (AdamW fix)?**  
"PyTorch's AdamW natively entangles weight decay uniformly across all `Parameters()`. If it decays the sparsity logic gates, the network aggressively starts severing connections arbitrarily instead of mathematically responding to the gradient loss. I explicitly partitioned parameter groups to protect the logic gates from arbitrary decay drop-off."

**What is the Pareto Frontier?**  
"It is a multi-objective optimization boundary. A configuration sits on the 'Pareto Frontier' when it is mathematically impossible to improve one metric (Sparsity) without directly degrading another (Accuracy). The Agent searches exclusively along this curve."

**Why a RAG pipeline?**  
"I didn't want static log files. By routing experiment outputs into FAISS vector embeddings, the system transforms numerical telemetry into semantic, retrievable intelligence. The user can literally ask the dashboard complex deployment questions intuitively."
